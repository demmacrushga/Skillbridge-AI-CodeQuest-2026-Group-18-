# Implementation Plan: Skill Gap Frontend Screens

**Branch**: `002-skill-gap-frontend` | **Date**: 2026-06-25
**Spec**: `specs/002-skill-gap-frontend/spec.md`

---

## Summary

Add a **Skills tab** to the SkillBridge AI Expo app with two screens:
1. **SkillGapScreen** — CV upload form + past reports list
2. **GapReportScreen** — Full ranked gap report with recommendations

The frontend calls the skill-gap-service (port 8003 via nginx `/skill-gap/`) using `expo-document-picker` to pick PDF/DOCX files and `FormData` + `fetch` to upload multipart requests. Navigation uses expo-router v6 with a nested Stack inside the new Skills tab.

## Design Decisions

- **Nested Stack inside Skills tab**: `gap-report/_layout.tsx` is a Stack inside the tab — keeps tab bar visible on the list screen and hides it on the detail screen without touching the root layout.
- **Fetch report by ID on GapReportScreen mount**: Supports deep links and prevents stale navigation state. The `analyseCV` response passes `reportId` via `router.push(`/gap-report/${reportId}`)`.
- **No `Content-Type` header on multipart uploads**: React Native's `fetch` sets the multipart boundary automatically. Manually setting `Content-Type: multipart/form-data` breaks the boundary encoding.
- **`copyToCacheDirectory: true` in DocumentPicker**: Required on Android — content:// URIs are not directly readable by `fetch` without copying to cache first.
- **Recommendation icons from `type` string**: Static `RESOURCE_TYPE_CONFIG` map — no enum coupling. Unknown types fall back to `link-outline`.

## Phased Task List

### Phase 1 — Setup (Blocking all screens)

- [ ] T001 Install `expo-document-picker` — run `cd frontend && npx expo install expo-document-picker` from repo root
- [ ] T002 [P] Create `frontend/types/skillGap.ts` — three TS interfaces: `RecommendationResponse { id, type, title, url: string | null }`, `SkillGapItem { id, skillName, importanceRank, description, recommendations }`, `GapReport { reportId, targetRole, gaps }`
- [ ] T003 [P] Create `frontend/services/skillGap.ts` — three functions following the `services/career.ts` pattern: (1) `analyseCV(token, asset, targetRole)` — builds `FormData` with file `{ uri, name, mimeType }` and `targetRole`, POSTs to `/skill-gap/analyse` WITHOUT setting `Content-Type`; (2) `getReports(token)` — GET `/skill-gap/reports`; (3) `getReport(token, reportId)` — GET `/skill-gap/reports/${reportId}`; all throw `{ status, message }` on non-ok responses

### Phase 2 — Navigation Wiring (Requires T001)

- [ ] T004 Create `frontend/app/(app)/gap-report/_layout.tsx` — `<Stack screenOptions={{ headerShown: false }} />`; this creates the Stack navigator for the nested gap-report route
- [ ] T005 Update `frontend/app/(app)/_layout.tsx` — add a 5th `<Tabs.Screen>` with `name="skill-gap"`, `title="Skills"`, icon `analytics-outline`/`analytics`; also add a screen entry for `gap-report` with `href: null` to hide it from the tab bar

### Phase 3 — Gap Report Detail Screen (Requires T002, T003, T004)

- [ ] T006 Create `frontend/app/(app)/gap-report/[reportId].tsx` — uses `useLocalSearchParams` to get `reportId`; calls `getReport(token, reportId)` on mount; renders: (a) custom header row with back button (`router.back()`) and `targetRole` + gap count; (b) `ScrollView` with one card per `SkillGapItem` sorted by `importanceRank`; each card shows rank badge (`#N`), `skillName`, `description`, horizontal divider, then `RecommendationRow` for each recommendation; `RecommendationRow` shows type icon from `RESOURCE_TYPE_CONFIG`, `title`, and a tappable link button (`Linking.openURL(url)`) hidden when `url` is null; loading and error states handled

### Phase 4 — Skill Gap Tab Screen (Requires T002, T003, T005, T006)

- [ ] T007 Create `frontend/app/(app)/skill-gap.tsx` — two sections in a `ScrollView`:
  - **Upload section** (top): file picker card (calls `DocumentPicker.getDocumentAsync({ type: ['application/pdf', 'application/vnd.openxmlformats-officedocument.wordprocessingml.document'], copyToCacheDirectory: true })`); shows selected file name once picked; `TextInput` for `targetRole` with `placeholder="e.g. Backend Developer"`; "Analyse CV" `TouchableOpacity` disabled when `!pickedFile || !targetRole.trim()`; on press: sets `isAnalysing=true`, calls `analyseCV`, on success navigates to `router.push(`/gap-report/${report.reportId}`)`, on error sets `analyseError`; loading state replaces upload section with pulsing card using `usePulse()` animation
  - **History section** (bottom): calls `getReports(token)` on mount; renders `SectionHeader` "Past Analyses"; maps `reports` to `ReportRow` (targetRole, formatted date, gap count, "View" button → `router.push`); empty state card when `reports.length === 0`
