# Implementation Plan: Delete Past Analysis

**Branch**: `feat/delete-analysis` | **Date**: 2026-06-26  
**Spec**: `specs/003-delete-analysis/spec.md`

## Summary

Add the ability for users to delete their own gap analysis reports. Backend adds `DELETE /skill-gap/reports/{reportId}` with ownership enforcement and cascade cleanup of all child rows and the associated CV upload record. Frontend adds a delete button to each row in the "Past Analyses" list (US1) and to the report detail header (US2), with a confirmation dialog and optimistic state removal.

## Technical Context

**Language/Version**: Java 21, Spring Boot 3.3.6 (backend) · TypeScript, Expo SDK 51 (frontend)

**Primary Dependencies**: Spring Data JPA (cascade delete), Spring Security (JWT), React Native `Alert` API

**Storage**: PostgreSQL 16, `skill_gap` schema — no migration needed

**Testing**: `./mvnw test` (JaCoCo ≥ 70%), Expo dev client for UI

**Target Platform**: Spring Boot service on port 8003 · Expo/React Native mobile app

**Project Type**: Microservice (backend) + Mobile app (frontend)

**Performance Goals**: DELETE endpoint should respond in < 200ms under normal load

**Constraints**: Ownership must be enforced — a user cannot delete another user's report. Response must not reveal report existence to other users (always 404 on miss).

**Scale/Scope**: Single delete operation; no bulk delete in v1.
