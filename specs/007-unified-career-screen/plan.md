# Implementation Plan: Unified Career Screen

**Branch**: `feat/unified-career-screen` | **Date**: 2026-06-29 | **Spec**: `specs/007-unified-career-screen/spec.md`

---

## Summary

Merge two separate tabs — **Careers** (`career-paths.tsx`) and **Roadmap** (`roadmap.tsx`) — into a single **Career** tab (`career.tsx`). The screen is state-driven: when no roadmap exists it renders the career path picker wizard; when a roadmap exists it renders the roadmap view directly. A "Change Path" action in the roadmap header transitions back to the picker in-place. The tab bar shrinks from 6 tabs to 5. No backend changes.

## Screen State Machine

```
                    Tab opens → fetchRoadmap()
                              │
                 404/null ────┤──── roadmap data
                    │                    │
             PICKER VIEW          ROADMAP VIEW
            (no roadmap)          (has roadmap)
                    │                    │
        generate succeeds         "Change Path" tapped
        → fetchRoadmap()          → setChangingPath(true)
                    └──────────────────┘
                     (both arrive at roadmap view)
```

`changingPath: boolean` forces picker even when `roadmap !== null`. New generation → `fetchRoadmap()` → `setChangingPath(false)`. Cancelled → `setChangingPath(false)`.

## Implementation Notes

### career.tsx composition

The file lifts directly from the two existing screens:

1. **Shared config**: `TYPE_CONFIG`, `PATH_META`, `LEVELS`, `GENERATING_STEPS` — merged at top.
2. **Picker section** (from `career-paths.tsx`): `GeneratingOverlay`, `SkillTags`, picker JSX + styles.
3. **Roadmap section** (from `roadmap.tsx`): `CompletedRow`, `UpNextCard`, `MilestoneCard`, roadmap JSX + styles.

Main component `CareerScreen` renders picker or roadmap based on `roadmap` state and `changingPath` flag.

### Change Path action (roadmap header addition)

```tsx
Alert.alert(
  'Change Career Path',
  'Your current roadmap will be replaced when you generate a new one.',
  [
    { text: 'Cancel', style: 'cancel' },
    { text: 'Change Path', style: 'destructive', onPress: () => setChangingPath(true) },
  ]
);
```

After new generation: `await generateRoadmap(...)` → `await fetchRoadmap()` → `setChangingPath(false)`.

### _layout.tsx changes

Replace career-paths + roadmap tab entries with a single career entry. Keep old names registered as `href: null` until files are deleted to avoid Expo Router warnings.

### index.tsx changes

All three navigation targets updated: `./roadmap` → `./career`, `./career-paths` → `./career`.
